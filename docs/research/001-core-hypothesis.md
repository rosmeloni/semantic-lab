# Core Hypothesis

## Working hypothesis

Existing enterprise platforms increasingly support the creation, storage, management and consumption of enterprise semantics.

They may provide capabilities such as:

- metadata harvesting
- business glossaries
- semantic models
- lineage
- active metadata
- context engineering
- semantic search
- automated documentation
- governed context for AI agents

However, the current working hypothesis of Semantic Lab is that few, if any, platforms treat **semantic governance itself** as a continuously reasoning, evidence-based and multi-agent process.

In such a system, enterprise semantics would not be maintained only through periodic human curation or static governance workflows. Specialised AI agents would continuously observe organisational artefacts, identify semantic changes or conflicts, gather supporting evidence, propose interpretations and request human decisions where authority or judgement is required.

## What this means

The proposed system would treat enterprise semantics as a living and evolving organisational capability.

It would support a continuous lifecycle:

1. Observe changes across organisational artefacts.
2. Extract semantic evidence.
3. Detect new concepts, inconsistencies and semantic drift.
4. Distinguish genuine conflicts from legitimate contextual differences.
5. Generate one or more candidate interpretations.
6. Explain each proposal using traceable evidence.
7. Assess confidence, risk and downstream impact.
8. Escalate decisions to the appropriate human owner.
9. Version and publish approved semantic changes.
10. Learn from previous governance decisions.

## Core distinction

Most existing systems appear to focus primarily on one or more of the following:

- creating semantic definitions
- storing and organising metadata
- exposing governed context
- enabling humans or AI agents to consume semantic information
- monitoring changes in technical assets

Semantic Lab investigates a different emphasis:

> Can semantic governance itself become a continuously operating reasoning process?

The semantic layer is therefore not only an asset to be queried or consumed. It is also an object that must be continuously interpreted, challenged, reconciled and evolved.

## Primary research question

> Can enterprise semantic governance be implemented as a continuously reasoning, evidence-based, multi-agent process while preserving human authority, accountability and legitimate bounded contexts?

## Supporting research questions

- How can candidate business concepts be inferred from SQL, BI models, APIs, documentation, tickets and source code?
- How can a system distinguish contradictory definitions from valid domain-specific variants?
- What evidence is sufficient to support a semantic change proposal?
- How should specialised agents challenge or validate one another's interpretations?
- Which decisions may be automated, and which require human approval?
- How can semantic changes be versioned, tested and rolled back?
- How can the system evaluate downstream impact before a semantic change is published?
- Can the system learn from previous human approvals, modifications and rejections?
- How should uncertainty and disagreement be represented rather than hidden?

## Current evidence

Early market research suggests that several platforms are moving toward:

- active metadata
- semantic bootstrapping
- context engineering
- semantic observability
- AI-assisted glossary generation
- governed semantic context for AI agents

This provides partial validation of the problem and confirms that the broader market is moving toward more dynamic semantic systems.

However, it remains unclear whether current products provide an integrated process that combines:

- continuous observation
- cross-artifact semantic reasoning
- conflict reconciliation
- evidence accumulation
- multi-agent review
- human-governed approval
- semantic versioning
- impact analysis
- learning from governance decisions

This hypothesis therefore remains under investigation.

## Falsification criteria

The hypothesis would be weakened or disproved if research identifies an existing product or research system that demonstrably:

- continuously observes heterogeneous enterprise artefacts;
- identifies and interprets semantic changes;
- distinguishes errors from legitimate bounded contexts;
- generates evidence-backed reconciliation proposals;
- uses specialised agents for semantic-maintenance responsibilities;
- manages approval, versioning and rollback;
- learns from governance outcomes;
- and operates as an integrated enterprise capability rather than as a collection of separate tools.

## Current status

**Status:** Under investigation

The purpose of the research programme is not to prove that the hypothesis is correct, but to test where it holds, where it fails and which parts may still represent genuine research or product opportunities.
